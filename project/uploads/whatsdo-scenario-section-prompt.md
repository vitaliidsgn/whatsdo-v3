# Новая секция: анимированный сценарий "How a booking happens"

Добавить новую секцию ниже бегущей строки индустрий (industries marquee). 
Секция показывает один проработанный сценарий бронирования от запроса 
пользователя до подтверждения. Это главная "story-секция" лендинга — 
она показывает, как WhatsDo работает на самом деле.

## Задача секции

За 14 секунд показать бизнес-владельцу:
1. Как клиент обращается к AI с запросом.
2. Что делает WhatsDo между запросом и подтверждением.
3. Как клиент получает confirmation.

Главная мысль: **WhatsDo — невидимый слой между AI и существующими 
системами бизнеса (Square, Stripe и др.), а не ещё один дашборд.**

## Layout секции

### Фон секции
- **Градиент** — мягкий тёплый или нейтральный, в палитре сайта 
  (#F6EFE5 как основа, к низу темнее или светлее с лёгким радиальным 
  свечением по центру).
- **НЕ использовать фотографии** конкретных индустрий (ресторан, 
  барбершоп, отель). Бегущая строка индустрий выше уже несёт посыл 
  "работаем со всеми" — здесь не нужно его разбавлять.
- Опционально: тонкая subtle-текстура (noise или dotted grid 5% opacity) 
  для глубины.

### Композиция
- Заголовок секции по центру сверху.
- Подзаголовок под ним.
- Большой мобильный мокап чата по центру (ниже).
- Подпись под мокапом (на финале анимации).
- Кнопка "↻ Replay" в нижнем правом углу.

### Заголовок секции
**H2:** "How a booking happens."
**Sub:** "From customer request to a paid booking in your existing 
system — in under two seconds."

### Мокап чата
Один мобильный фрейм (iOS-style, минималистичный чёрный фрейм с notch). 
Размер — крупный, по центру секции, занимает большую часть высоты 
viewport. Внутри фрейма — интерфейс AI-чата.

**Шапка чата:**
- Иконка sparkle (✨) в зелёном круге.
- "AI Assistant"
- "Powered by WhatsDo" мелким серым ниже.

**Тело чата** — сообщения появляются последовательно по сценарию ниже.

## Сценарий анимации (полный, по секундам)

Длина одного цикла — **14 секунд**. После — анимация встаёт на 
финальном кадре. **Не зацикливается.**

### 00:00 — стартовое состояние
Тело чата пустое. Виден только header.

### 00:00 → 00:01.5 — пользователь печатает запрос
Снизу справа всплывает чёрный bubble:
> "I need a good barber in downtown"

- Bubble: translate Y 16px → 0, opacity 0 → 1, 200ms ease-out.
- Текст печатается ~40ms/символ.

### 00:01.5 → 00:02.5 — AI typing indicator
Слева под bubble юзера: иконка sparkle + три прыгающие точки.
Длительность: ровно 1 секунда.

### 00:02.5 → 00:04 — AI отвечает текстом
Bubble слева:
> "I found a highly-rated spot near you with availability today:"

- Появление + типинг текста.

### 00:04 → 00:04.5 — карточка бизнеса появляется
Под текстом ответа вылетает карточка:

```
┌────────────────────────────────────┐
│ Classic Cuts Barbershop    ⭐ 4.8  │
│ Barbershop · $$ · 0.4 mi           │
│                                    │
│ [2:00 PM]  [3:30 PM]  [4:15 PM]   │
│                                    │
│        [Book Now →]                │
└────────────────────────────────────┘
```

- Анимация: scale 0.95 → 1, opacity 0 → 1, translate Y 16px → 0, 
  400ms ease-out с лёгким overshoot.

### 00:04.5 → 00:05.5 — слот выбирается
Time-слот "3:30 PM" подсвечивается зелёной заливкой с лёгкой 
пульсацией (scale 1 → 1.05 → 1, 300ms).

### 00:05.5 → 00:06 — кнопка нажимается
Кнопка "Book Now" сжимается (scale 1 → 0.97 → 1, 150ms), затем 
лейбл меняется на "Booking…" со спиннером.

### 00:06 → 00:09 — КЛЮЧЕВОЙ КАДР: WhatsDo делает работу

Над или под карточкой бизнеса появляется **встроенный в чат 
статус-блок** — это единственное место, где виден сам WhatsDo. 
Не отдельный экран, а компактная техническая лента.

```
┌────────────────────────────────────┐
│  ⚡ Processing via WhatsDo    1.8s │
│                                    │
│  ●  Checking availability          │
│  ●  Holding the slot               │
│  ●  Charging $45 via Stripe        │
│  ●  Syncing with Square POS        │
│                                    │
└────────────────────────────────────┘
```

**Тайминг строк:**
- 00:06.0 — появляется "Checking availability"
- 00:07.0 — появляется "Holding the slot"
- 00:08.0 — появляется "Charging $45 via Stripe"
- 00:08.7 — появляется "Syncing with Square POS"

**Анимация каждой строки:**
- Появление: translate X -10px → 0, opacity 0 → 1, 200ms.
- Точка слева сначала серая (●), через 300ms становится зелёной (✓) — 
  операция выполнена.

К 00:09 все четыре строки на месте, все галочки зелёные.

В правом верхнем углу блока — мелкий timer-счётчик "1.8s" появляется 
на финале (на 00:09).

**Этот блок — главный визуальный момент анимации.** Он показывает:
- Что делает WhatsDo конкретно (4 операции).
- Что мы интегрируемся, а не заменяем существующие системы 
  ("Charging via Stripe", "Syncing with Square POS").
- Скорость работы (1.8s).

### 00:09 → 00:09.7 — статус-блок сворачивается
Блок "Processing via WhatsDo" схлопывается в одну компактную строку:
> **✓ Processed via WhatsDo · 1.8s**

- Анимация: высота блока collapse to 1 line, 500ms ease-in-out.

### 00:09.7 → 00:11 — финальный bubble подтверждения от AI
Слева появляется AI-bubble:
> "Booked. You're confirmed for 3:30 PM today at Classic Cuts. 
> Confirmation sent to your email and to the shop's calendar."

- Появление + типинг ~30ms/символ.

### 00:11 → 00:12 — мини-карточка confirmation
Под последним bubble выпрыгивает компактная плашка:

```
┌────────────────────────────────────┐
│ ✓ Booking #BR-7841                 │
│   Today · 3:30 PM                  │
│   $45 paid · Stripe                │
│   Synced to shop's calendar        │
└────────────────────────────────────┘
```

- Scale-up появление, 300ms ease-out.

### 00:12 → 00:14 — финальный кадр стоит
Анимация останавливается. На экране видно:
- Запрос юзера сверху
- Ответ AI с карточкой барбера (карточка остаётся видна, слот 3:30 PM 
  подсвечен зелёным)
- Свёрнутая строка "✓ Processed via WhatsDo · 1.8s"
- Финальное подтверждение от AI
- Карточка booking confirmation

### 00:13 — подпись под мокапом
Fade-in 400ms:
> *Same flow for every service: barbershops, restaurants, hotels, 
> clinics, studios. Every booking lands in your existing system.*

### Replay-ссылка
В нижнем правом углу секции — мелкая ссылка с иконкой:
> **↻ Replay**

При клике — анимация проигрывается заново с 00:00.

## Технические требования

**Easings:**
- Появления элементов: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out 
  с лёгким overshoot).
- Исчезновения: `cubic-bezier(0.4, 0, 1, 1)` (быстрый ease-in).

**Производительность:**
- Все анимации только через `transform` и `opacity`. 
  Никаких `top/left/width/height`.
- Использовать `will-change: transform` на анимируемых элементах.

**Триггер:**
- Анимация запускается, когда секция попадает во вьюпорт через 
  Intersection Observer (threshold 0.5).
- Запускается **один раз за визит**. Если пользователь скроллил мимо 
  и вернулся — ничего не делать (только Replay-кнопкой).

**Доступность:**
- При `prefers-reduced-motion: reduce` — анимация не проигрывается. 
  Сразу показывается финальный кадр всех элементов.
- Replay-кнопка работает в любом случае.

**Звук:** нет.

**Адаптив:**
- Desktop: мокап телефона по центру, размер ~380x780px.
- Mobile: мокап на всю ширину viewport с padding 24px по бокам, 
  высота автоматическая.

## Что НЕ делать

- НЕ показывать отдельный "Business dashboard" или интерфейс WhatsDo 
  как платформы. WhatsDo не является дашбордом — он слой.
- НЕ использовать фотографию ресторана или барбершопа на фоне. 
  Только градиент или нейтральная текстура.
- НЕ зацикливать анимацию. После 14 секунд — стоп, финальный кадр.
- НЕ менять барбершоп на ресторан внутри сценария. Бегущая строка 
  индустрий выше уже работает на разнообразие.
- НЕ использовать реальные названия известных брендов в карточке 
  бизнеса (никаких реальных названий ресторанов/салонов SF). 
  "Classic Cuts Barbershop" — выдуманное имя, оставить.
- НЕ ставить логотипы Square/Stripe в виде картинок без подтверждения 
  правомерности использования. В первой итерации — упоминание 
  текстом ("via Stripe", "Square POS") без логотипов.

## Что менять не нужно

- Hero выше (preheader, H1, sub, кнопки, tagline).
- Бегущая строка индустрий.
- Шапка с навигацией.
- Цветовую палитру.
- Все секции ниже этой (если есть).
